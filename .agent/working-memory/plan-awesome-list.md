**Target page:** `/resources/awesome-modeling-practices/` (new section, or under `/standards/`)

---

## Approach: Hugo build-time remote fetch (`resources.GetRemote`)

Hugo 0.91+ can fetch a remote resource (URL) at `hugo build` time and embed/render it
as part of the static site. This project uses Hugo 0.159.1, so the feature is available.

**Why this approach:**
- No JavaScript or client-side fetching required
- Content is always fresh on each build / deploy
- Works inside the Docker container build with no extra tooling
- Fully static output, no runtime dependencies

**Trade-off:** The page only updates when the site is rebuilt. For a GitHub Pages
site this is acceptable; add a scheduled CI trigger (e.g., weekly `workflow_dispatch`)
to keep it current.

## Implementation steps

### Step 1 — Content page

Create `content/en/resources/awesome-modeling-practices/_index.md`:

```markdown
---
title: "Awesome Modeling Practices"
linkTitle: "Awesome Modeling Practices"
description: "A curated, community-maintained list of resources for good modeling practices — maintained by the OMF Early Career Scholars Working Group."
layout: "awesome-list"
menu:
  main:
    weight: 25
---
```

The `layout: "awesome-list"` front-matter key triggers a custom layout (step 2).

**Navigation note:** If the site grows a dedicated "Resources" top-level section,
this page belongs there. For now it can live under `contribute/` or as a standalone
top-level page. Discuss with maintainers.

---

### Step 2 — Custom layout

Create `layouts/_default/awesome-list.html` (overrides the default single/list
layout specifically when `layout: awesome-list` is declared in front matter):

```html
{{ define "main" }}
  {{ $rawURL := "https://raw.githubusercontent.com/openmodelingfoundation/awesome-modeling-practices/main/README.md" }}
  {{ $opts := dict "headers" (dict "User-Agent" "Hugo/OMF-Site") }}
  {{ with resources.GetRemote $rawURL $opts }}
    {{ with .Err }}
      {{ errorf "awesome-list: failed to fetch README: %s" . }}
    {{ else }}
      {{/* Fix relative links → absolute GitHub links */}}
      {{ $content := .Content }}
      {{ $content = $content | replaceRE `\(CONTRIBUTING\.md\)` "(https://github.com/openmodelingfoundation/awesome-modeling-practices/blob/main/CONTRIBUTING.md)" }}
      {{/* Drop the H1 title (it's in the page <h1> already) */}}
      {{ $content = $content | replaceRE `(?m)^# Awesome Modeling Practices\n` "" }}
      <div class="awesome-list-content td-content">
        {{ $content | markdownify }}
        <p class="awesome-list-source">
          Source: <a href="https://github.com/openmodelingfoundation/awesome-modeling-practices" target="_blank" rel="noopener">
            openmodelingfoundation/awesome-modeling-practices
          </a> — auto-imported at build time.
        </p>
      </div>
    {{ end }}
  {{ else }}
    <div class="alert alert-warning">
      The Awesome Modeling Practices list could not be loaded. Please visit
      <a href="https://github.com/openmodelingfoundation/awesome-modeling-practices">the repository</a> directly.
    </div>
  {{ end }}
{{ end }}
```

**Key transformations applied before `markdownify`:**
| Problem | Fix |
|---|---|
| Relative link `CONTRIBUTING.md` | Replace with absolute GitHub blob URL |
| Duplicate `# Awesome…` H1 | Strip (page already has its own `<h1>` from Docsy) |
| Badge `[![Awesome](...)](...)` | No change needed — renders correctly as `<img>` |
| Internal anchor links (`#modeling-principles`) | No change needed — Goldmark generates matching IDs |

### Step 3 — SCSS polish

Add to `assets/scss/_styles_project.scss`:

```scss
// ── Awesome Modeling Practices list ────────────────────────────────────────

.awesome-list-content {
  // Section dividers (## headings)
  h2 {
    margin-top: 2rem;
    padding-bottom: 0.4rem;
    border-bottom: 3px solid $primary;
    font-size: 1.4rem;
    color: $primary;
  }

  // Table of contents auto-generated by the README
  > ul:first-of-type {
    display: flex;
    flex-wrap: wrap;
    gap: 0.4rem;
    list-style: none;
    padding-left: 0;

    li a {
      display: inline-block;
      padding: 0.25rem 0.75rem;
      border-radius: 2rem;
      background: $secondary;
      color: #fff;
      font-size: 0.85rem;
      text-decoration: none;

      &:hover {
        background: $primary;
        text-decoration: none;
      }
    }
  }

  // Resource entries (bullets inside each section)
  ul li {
    margin-bottom: 0.6rem;
    line-height: 1.5;

    // Resource name (the link)
    a:first-child {
      font-weight: 600;
    }

    // Separator dash character between name and description
    &::marker {
      color: $primary;
    }
  }

  // "Source: …" attribution line
  .awesome-list-source {
    margin-top: 2.5rem;
    font-size: 0.8rem;
    color: $gray-600;
    border-top: 1px solid $gray-300;
    padding-top: 0.5rem;
  }
}
```

`$primary`, `$secondary`, and `$gray-*` are already defined by Bootstrap / Docsy
so no new variables are needed.

**Optional enhancement (cards layout):** Each `##` section can be wrapped in a
Bootstrap card by doing a post-`markdownify` string replacement in an extra partial,
but this requires more complex regex and is harder to maintain. The styled-list
approach above achieves a polished look with minimal coupling to the README structure.

### Step 5 — Cache / freshness strategy

Hugo caches `resources.GetRemote` results in `resources/_gen/`. For GitHub Pages:

1. **Don't commit `resources/_gen/` to git** — already git-ignored by default in
   Hugo projects.  Confirm `.gitignore` covers `resources/_gen/`.
2. **Scheduled CI rebuild** — add a weekly `schedule:` trigger to
   `.github/workflows/` so the page auto-updates even without a content push.
3. **Manual cache bust** — run `docker compose run --rm hugo --ignoreCache` to
   force a fresh fetch during local development.

## Risks and mitigations

| Risk | Mitigation |
|---|---|
| Remote fetch times out during build | Hugo respects `HUGO_HTTPCLIENT_TIMEOUT`; set to 10s in CI env |
| README structure changes break CSS selectors | CSS targets generic `h2`/`ul`/`li` — resilient to content changes |
| GitHub rate-limits unauthenticated fetches | `resources.GetRemote` includes caching; add `GITHUB_TOKEN` as an `Authorization` header env var in CI if needed |
| Relative links in README (other than `CONTRIBUTING.md`) | Audit README for other relative links before launch; add `replaceRE` rules as needed |
